# Codex for Obsidian PRD

## 1. 一句话目标

在 Obsidian 右侧栏嵌入一个接近 Codex 桌面端体验的聊天工作台，让 Codex CLI 直接以当前 Obsidian vault 为工作区，帮助用户阅读、整理、修改和管理个人知识库。

## 2. 背景

用户已经在本机登录并配置 Codex CLI，也已经在 Obsidian 中使用类似 Claudian 的侧栏式 Agent UI。现在需要一个独立 Obsidian 插件：

- 不要求再次登录或配置 API key。
- 不要求手动选择工作目录。
- 不复用 Claudian 源码，但可以参考其 UI 信息架构。
- 所有设置和错误提示默认中文。

## 3. 目标用户

- 用户本人。
- 使用场景是桌面端 Obsidian。
- 用户不需要理解 CLI、JSON-RPC、MCP、sandbox 等技术细节。

## 4. 核心场景

1. 用户在 Obsidian 右侧栏打开 Codex。
2. 用户直接提问，例如“帮我整理今天的日记”。
3. 插件以当前 vault 为工作区启动 Codex。
4. Codex 读取、搜索、修改 vault 内文件。
5. 用户能看到消息、计划、命令、文件变更、图片和上下文用量。
6. 用户可以通过 `/` 调用 skills，通过 MCP 使用已配置工具。

## 5. 第一版范围

### 5.1 侧栏聊天

- 插件提供右侧栏视图。
- 支持多会话标签：
  - 新建会话。
  - 切换会话。
  - 重命名会话。
  - 右键会话标签可删除会话。
  - 关闭后恢复。
- 支持流式消息展示。
- 支持中断当前回复。

验收：

- Obsidian 能打开右侧栏。
- 发送普通文本后能收到 Codex 回复。
- 重启 Obsidian 后能恢复会话列表。

### 5.2 Codex app-server 接入

- 插件通过 `codex app-server --listen stdio://` 接入 Codex。
- 读取并复用当前 Codex：
  - 登录状态。
  - 模型列表。
  - 用户配置。
  - skills。
  - MCP 状态。
- 不在插件中要求填写 OpenAI key。
- app-server 进程异常时显示中文错误，可手动重启。
- skills/MCP 读取需要单独超时，不能阻塞主聊天。

验收：

- 插件能显示 Codex 已登录状态。
- 能读取模型列表。
- 能读取 skills。
- 能读取 MCP 服务器状态。

### 5.3 默认工作区

- 默认工作区为当前 Obsidian vault。
- 默认目标 vault 为当前打开的 Obsidian vault。
- 用户无需手动设置工作目录。

验收：

- Codex 新建或修改文件时，默认落在当前 vault 内。

### 5.4 权限

- 默认权限为 `workspace-write`。
- 默认只允许当前 vault 和必要临时目录写入。
- UI 显示当前权限：
  - 只读。
  - 工作区可写。
  - 完全放开。
- 如果 Codex 请求额外权限，插件用中文弹窗询问。

验收：

- 默认不是全盘可写。
- 权限请求能被批准或拒绝。

### 5.5 模型、速度、思考强度、模式

- 底部工具栏支持：
  - 模型选择。
  - 速度选择。
  - 思考强度选择。
  - Agent / Plan 模式切换。
- 选择项要真实传给 `turn/start`。

验收：

- 模型选择改变后，新 turn 使用所选模型。
- Plan 模式能让 Codex 输出计划。

### 5.6 文件和图片

- 支持添加 vault 内文件作为上下文。
- 支持拖拽图片或选择图片。
- 用户消息中显示图片缩略图。
- Codex 回复中的图片路径或图片消息可渲染。
- 图片可点击放大。

验收：

- 发送带图片的消息后，Codex 能收到图片输入。
- 回复中出现图片时，侧栏能显示。

### 5.7 消息渲染

- 支持渲染：
  - Markdown。
  - 代码块。
  - 表格。
  - diff。
  - 命令输出。
  - 文件变更。
  - 工具调用。
  - 计划块。
  - 推理摘要。
  - 图片。
- 思考和执行过程需要像 Codex 桌面端一样可见：
  - 思考中呼吸态。
  - 思考完成时间。
  - 命令输出。
  - 文件编辑记录。
  - MCP 工具过程。
- 长内容不能撑破侧栏。

验收：

- 代码块可复制。
- 工具调用可折叠。
- 长命令输出可滚动或折叠。

### 5.8 Slash skills

- 输入 `/` 时弹出 skills 列表。
- 支持搜索。
- 选择后作为 Codex `skill` 输入发送，而不是普通文本。

验收：

- `/` 能列出本机可用 skills。
- 选择 skill 后 Codex 按该 skill 行为执行。

### 5.9 MCP

- 插件显示 Codex 当前 MCP 服务器状态。
- 支持查看服务器名、工具数量、授权状态。
- 支持 MCP OAuth 登录入口。
- 支持 MCP elicitation 表单。
- 不在插件中编辑 MCP 配置；MCP 配置仍由 Codex CLI 管理。

验收：

- 能看到当前 MCP 状态。
- MCP 需要用户输入时，Obsidian 弹出中文表单。

### 5.10 中文设置页

- 设置页全中文。
- 设置项包括：
  - Codex CLI 路径检测。
  - 默认模型。
  - 默认速度。
  - 默认思考强度。
  - 默认权限。
  - 默认 Agent / Plan 模式。
  - 是否自动打开侧栏。
  - 是否显示上下文容量。
- 设置只保存插件偏好，不保存 OpenAI 密钥。

验收：

- 重启 Obsidian 后设置仍生效。
- 页面没有英文主文案。

## 6. 非目标

- 不做移动端。
- 不做远程服务。
- 不做 Codex 全局配置编辑器。
- 不保存 OpenAI key。
- 不复用 Claudian 源码。
- 不在第一版实现语音输入。
- 不在第一版实现 Codex Cloud 任务浏览。

## 7. 技术原则

- 用 TypeScript 开发 Obsidian 插件。
- Codex 相关逻辑集中封装，避免 app-server 协议变动影响 UI。
- UI 使用 Obsidian CSS 变量，兼容明暗主题。
- 主聊天功能优先，skills/MCP 状态失败不能让聊天不可用。
- 错误要用产品经理能看懂的中文表达。

## 8. 用户 8 条需求覆盖

1. 侧边栏对话：覆盖，见 5.1。
2. 模型、速度、思考、权限、模式、文件、上下文容量：覆盖，见 5.5、5.6。
3. `/` 唤起 skills：覆盖，见 5.8。
4. 支持 skills、MCP：覆盖，见 5.8、5.9。
5. 图片渲染、接近 Codex 桌面端：覆盖，见 5.6、5.7。
6. 不额外配置、复用 Codex 登录：覆盖，见 5.2、5.10。
7. 默认当前 Obsidian 仓库：覆盖，见 5.3。
8. 插件设置页中文：覆盖，见 5.10。

## 9. Phase 通过标准

- Phase 1：本 PRD 覆盖 8 条需求，明确做与不做。
- Phase 2：测试用例覆盖 PRD 核心能力。
- Phase 3：Obsidian 能加载插件骨架。
- Phase 4：Codex app-server 后端可读账号、模型、skills、MCP。
- Phase 5：基础聊天闭环可用。
- Phase 6：消息渲染接近桌面端。
- Phase 7：底部工具栏参数真实生效。
- Phase 8：skills/MCP 基础交互可用。
- Phase 9：中文设置页和体验打磨完成。
- Phase 10：完成最终验收并部署。
